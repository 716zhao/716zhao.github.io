第一章：Spring框架概述：
1.目的：Spring是为了解决繁琐的软件开发而创建的。

2.作用：解耦合，方便测试，集成框架，降低API(对JDBC等进行封装)开发难度。

3.特点：Spring不同于Struts、Hibernate等单层框架，它可以糅合多个框架，将多个单层框架建立成一个连贯的整体，Spring提供了一个更完善的开发环境。

4.概述：轻量级(Jar包数量少)的，开源的JavaEE框架。

5.核心组成部分：IOC和Aop
(1)IOC:控制反转，把创建的对象过程交给Spring进行管理。
(2)Aop:面向切面，在不修改源代码的情况下添加或修改功能。
参考文献:https://baike.so.com/doc/6841943-7059231.html

第二章：IOC容器

  1. 什么是IOC：控制反转，把对象的创建和对象的调用过程，交给Spring管理.

  2. IOC底层原理：
    （1）xml解析、工厂模式、反射

  3. IOC过程：
    （例子：）
    第一步 xml配置文件，配置创建的对象

    第二步 有service类和dao类，创建工厂类
    calss UserFactory{

     Class clazz = Class.forName(classValue);        //通过反射创建对象

                             }

    }

4.IOC(接口)
1.IOC思想基于IOC容器完成，IOC容器底层就是对象工厂。（★）

2.Spring提供两种方式实现IOC容器（两个接口）
        （1）BeanFactory：IOC容器基本实现，是Spring内部的使用接口，不提供开发人员进行使用
    ★加载配置文件时不创建对象，在使用对象时才去创建
        （2）ApplicationContext：BeanFactory接口的子接口，提供更多功能，一般由开发人员进行使用
    ★加载配置文件时就创建对象

3.ApplicationContext的实现类：
    1.FileSystemXmlApplicationContext参数是 带盘符的路径，绝对路径
    
     2.ClassPathXmlApplicationContext参数是 src下的路径

第三章：IOC操作Bean管理
1.什么是Bean管理
①Spring创建对象
②Spring属性注入

2.两种Bean管理操作的实现方式
    ①通过xml配置文件实现
        1.创建对象：
        （默认执行无参方法  ps:类中写了有参构造器不写无参构造器，默认没有无参构造器）
        <bean id = "User" class = "con.atguigu.spring5.User"></bean>
        
        2.属性注入：
        DI(依赖注入)，是IOC的一种具体实现，即注入属性
        两种注入方式：
            ①.通过set方法注入（需要事先写好set方法）
                    <bean id="book" class="com.atguigu.spring5.testdemo.Book">
                            <property name="bname" value="康师傅"></property>
                      </bean>
            通过property(属性)操作


            ②.通过有参构造器注入
                     <bean id="book" class="com.atguigu.spring5.Book">
                            <constructor-arg name="bname" value="美年达"></constructor-arg>
                            <constructor-arg name="bauthor" value="百事可乐"></constructor-arg>
                        <constructor-arg index="?" value="?"></constructor-arg>
                    </bean>
            通过constructor-arg给属性赋值；不要无参构造器; 需要配置所有属性; 或者可以通过index选择为第?个属性赋值
            
            
            ③.p名称空间注入（了解即可，也需要提前在类中写好set方法）
                     第一步：在配置文件中添加p名称空间
                      xmlns:p="http://www.springframework.org/schema/p"
                     第二步：注入属性
                     <bean id="book" class="com.atguigu.spring5.Book" p:bname="美年达" p:bauthor="百事可乐"></bean>
            也需要无参构造器

        3.xml注入其他类型属性
          1.字面量：
             （1）null值

            <property name="美年达">
            <null/>
            </property>

               (2)  属性值包含特殊符号（如想显示书名号等等）

            1.转义符号lt...

            2.CDATA
            <property name="饮料名">
            <value=![CDATA<<百事可乐>>]></value>
            </property>

3.Spring中存在的两种类型bean
    ①.普通bean
        返回类型就是它在配置文件中定义的bean类型



        如上图⬆


    ②.工厂bean
        返回类型与配置文件中定义的bean类型不一致

        配置文件：
         <bean id="myBean" class="com.atguigu.spring5.factorybean.MyBean">
            </bean>

        返回类型:
                Course course = applicationContext.getBean("myBean",Course.class);
                System.out.println(course);

        如此区别的原因在于在MyBean类中实现了FactoryBean类，可以使用泛型同时与函数getObject返回其他的类型：
        @Override
            public Course getObject() throws Exception {
               Course course = new Course();
                course.setCname("aaa");
                return course;
            }


4.bean的作用域：
    ①.Spring中，bean实例存在单实例和多实例

    ②.Spring中，默认bean实例为单实例对象(通过多次创建对象，输出地址可验证。若两次地址一致则为单实例反之。)


    ③Spring中，可以设置bean实例为多实例对象
        <bean id="" class="" scope="singleton(单实例)/prototype(多实例)"></bean>


    ④singleton和prototype的区别
        第一：singleton在加载配置文件的时候就创建单实例对象，

            ApplicationContext applicationContext = new ClassPathXmlApplicationContext("bean1.xml");

                  prototype在调用getBean方法时才加载对象。
            
            Book book1 = applicationContext.getBean("book",Book.class);

5.Bean的生命周期：
    ①生命周期概念：对象从创建到销毁的过程。

    ②bean的生命周期：
        1. --->     第一步，构造器被执行了
            第二步，属性设置被调用了
            第三步，初始化函数被调用了(通过<bean init-mathod="？"></bean> 调用)
            第四步，获取创建的bean对象实例
            com.atguigu.spring5.Bean.bean@2038ae61
            第五步，销毁函数被调用了(通过<bean destory-mathod="？"></bean> 调用)

        2. --->     ①添加后置处理器后：
        
            第一步，构造器被执行了
            第二步，属性设置被调用了
            第三步，把bean实例传递给bean后置处理器的方法
            第四步，初始化函数被调用了
            第五步，把bean实例传递给bean后置处理器的方法
            第六步，获取创建的bean对象实例
            com.atguigu.spring5.Bean.bean@2038ae61
            第七步，销毁函数被调用了(通过<bean destory-mathod="？"></bean> 调用)
